Introduction

Hawk‚Äëi is an open security intelligence platform for Web3 repositories. It combines deterministic static analysis, AI reasoning, and live exploit simulation into a unified, privacy‚Äëfirst tool.

With the v0.7.0 release, Hawk‚Äëi evolves from a functional scanner to an audit‚Äëgrade security intelligence system. It now produces professional reports with quantified risk scores, guided remediation snippets, and optional telemetry to measure ecosystem impact.

Open source AI‚Äëaugmented Self‚Äëhostable v0.7.0

Installation

Prerequisites

  • Python 3.9+
  • Node.js 16+ (for sandbox)
  • Docker (optional, for exploit simulation)

via pip (recommended)

$ pip install hawki

Package available at pypi.org/project/hawki

from source

$ git clone https://github.com/0xSemantic/hawki.git
$ cd hawki
$ pip install -e .

Docker image

$ docker pull levichinecherem/hawki:latest
$ docker run --rm -v $(pwd):/repo levichinecherem/hawki scan /repo

Docker Hub: hub.docker.com/r/levichinecherem/hawki

Quickstart

Scan a local repository with default settings (Minimal mode):

$ hawki scan ./my-project

Run a full audit with AI reasoning and exploit simulation (Full Audit mode):

$ hawki scan ./contracts --ai --sandbox --format pdf --telemetry

Generate a report from the latest scan:

$ hawki report --format html --output report.html

View the security score of a previous findings file:

$ hawki score findings.json

Display local telemetry stats:

$ hawki metrics

For more, see the CLI reference.

CLI Reference

The main command is hawki. Below are all available subcommands and options.

hawki scan

Perform a one‚Äëtime security scan on a local directory or Git repository. Supports three operational modes depending on flags.

hawki scan <path> [options]

Options:

  • -v, --verbose ‚Äì Enable debug logging.
  • -o, --output-dir <dir> ‚Äì Directory to store reports (default: ./hawki_reports).
  • --ai ‚Äì Enable AI‚Äëpowered analysis (requires API key).
  • --ai-model <model> ‚Äì LLM model (e.g., gemini/gemini-1.5-flash, openai/gpt-4).
  • --api-key <key> ‚Äì API key for the chosen LLM (or set env var).
  • --sandbox ‚Äì Run exploit simulation sandbox (requires Docker).
  • --telemetry ‚Äì Opt in to anonymous usage metrics.
  • --format {md,json,html,pdf} ‚Äì Output report format (default: md).

hawki monitor

Continuously monitor a repository or deployed contract for changes and trigger automatic rescans.

hawki monitor [target] [options]

Options:

  • -c, --config <file> ‚Äì JSON configuration file for watchers.
  • --state-dir <dir> ‚Äì Directory to store watcher state (default: ~/.hawki/monitor_state).
  • --alert-log <file> ‚Äì File to append alerts to.
  • --interval <sec> ‚Äì Polling interval in seconds (default: 60).
  • --branch <name> ‚Äì Git branch to monitor (default: main).
  • --contract-address <addr> ‚Äì Ethereum contract address to monitor.
  • --rpc-url <url> ‚Äì RPC URL for contract monitoring (default: http://localhost:8545).

hawki report

Generate an audit‚Äëgrade report from a previous scan‚Äôs findings (JSON file) or from the latest scan.

hawki report [--input findings.json] [--format pdf|md|html|json] [--output report.pdf]

Options:

  • -i, --input <file> ‚Äì Path to findings JSON file (if omitted, uses latest scan in output dir).
  • -f, --format <format> ‚Äì Output format (md, json, html, pdf).
  • -o, --output <file> ‚Äì Output file path.

hawki score

Calculate and display the security score (0‚Äì100) for a given findings file.

hawki score findings.json

hawki metrics

Display locally stored telemetry statistics (if any).

hawki metrics

hawki simulate

Run a specific exploit simulation against a contract (advanced usage).

hawki simulate --target Vault.sol --attack reentrancy

Options:

  • --target <file> ‚Äì Solidity file or contract name.
  • --attack <name> ‚Äì Name of the attack script (e.g., reentrancy).

Audit‚ÄëGrade Reporting (ARS v2)

v0.7.0 introduces the Audit‚ÄëGrade Report System, producing professional reports suitable for grant applications, enterprise reviews, and auditor handoffs.

Report Structure

  • Executive Summary ‚Äì total contracts, severity counts, security score, risk classification, mode used.
  • Vulnerability Breakdown ‚Äì pie chart (severity) + bar chart (type) + fallback table.
  • Per‚ÄëFinding Details ‚Äì each finding includes:
    • Title & severity
    • File & line number
    • Vulnerable code snippet
    • Recommended fix snippet (from Remediation Engine)
    • Explanation (AI or fallback)
    • Impact analysis
    • Exploit reproduction steps (if sandbox succeeded)
  • Simulation Metrics ‚Äì success rate, balance deltas, gas used.

Output Formats

Markdown, JSON, HTML, and PDF (optional dependencies).

Security Score

Hawk‚Äëi computes a deterministic 0‚Äì100 security score based on findings and sandbox results.

Formula

Base: 100. Deductions per finding:

  • Critical: -15
  • High: -8
  • Medium: -4
  • Low: -1

If sandbox is enabled, each successfully reproduced exploit adds an extra -5 (capped).

Classification Bands

50‚Äì74Moderate Risk

0‚Äì24Critical Risk

The score is included in every report and can be viewed standalone with hawki score.

Guided Remediation Engine

Every finding now includes a context‚Äëaware fix snippet. The Remediation Engine uses templates and AST information to generate accurate, secure code replacements.

Example for reentrancy:

// Vulnerable
function withdraw() external {
    uint amount = balances[msg.sender];
    (bool success, ) = msg.sender.call{value: amount}("");
    require(success);
    balances[msg.sender] = 0;
}

// Recommended fix
function withdraw() external nonReentrant {
    uint amount = balances[msg.sender];
    balances[msg.sender] = 0;
    (bool success, ) = msg.sender.call{value: amount}("");
    require(success);
}

Fixes are populated automatically in reports and are also available via the hawki report command.

Telemetry (Opt‚ÄëIn)

To measure ecosystem impact, Hawk‚Äëi can collect anonymous usage metrics when the --telemetry flag is used.

Collected data includes:

  • Total scans performed
  • Findings per severity
  • Simulation success rate (if sandbox enabled)
  • Hawk‚Äëi version
  • Execution mode (minimal/enhanced/full)

No source code, repository names, or IP addresses are ever collected. Data is stored locally in ~/.hawki/metrics.json and can be viewed with hawki metrics.

If you opt in, Hawk‚Äëi may send aggregated statistics to a public endpoint to power the community metrics badge. You can disable telemetry entirely by never using the flag or setting HAWKI_TELEMETRY=0.

Monitoring

Hawk‚Äëi can continuously monitor repositories via git hooks, file watchers, or GitHub webhooks. Set up a webhook endpoint:

Configure your GitHub repo to send push events to http://your-host:8080/webhook.

Architecture overview

Hawk‚Äëi consists of seven core subsystems, plus new additions in v0.7.0:

  • RIE ‚Äì Repository Intelligence Engine
  • SRE ‚Äì Static Rule Engine (30+ rules)
  • AIRE ‚Äì AI Reasoning Engine
  • ESS ‚Äì Exploit Simulation Sandbox
  • AMS ‚Äì Activity Monitoring System
  • SDL ‚Äì Security Data Layer
  • DIL ‚Äì Dashboard & Interface
  • REM ‚Äì Remediation Engine (new)
  • TEL ‚Äì Telemetry Module (new)
  • ARS ‚Äì Audit Report System (new)

Each can be used independently; they communicate via internal APIs and a shared metadata store.

Subsystems in depth

Repository Intelligence Engine (RIE)

Traverses file systems, parses Solidity (via tree‚Äësitter, Slither AST), builds dependency graphs and contract registries.

Static Rule Engine (SRE)

Dynamically loads rules from rules/ directory. v0.7.0 includes 30 standardized rules, each with explanation, impact, and fix templates.

AI Reasoning Engine (AIRE)

Supports OpenAI, Anthropic, Gemini, and local LLMs. Now integrates with scoring and provides fallback explanations when AI is disabled.

Exploit Simulation Sandbox (ESS)

Dockerized Hardhat/Foundry environment. Returns detailed metrics (balances, gas, tx hash) and influences the security score.

Remediation Engine (REM)

Generates context‚Äëaware fix snippets using templates and AST information.

Telemetry Module (TEL)

Opt‚Äëin anonymous metrics collection and local storage.

Audit Report System (ARS)

Produces professional reports with charts, scores, and per‚Äëfinding remediation.

AI Integration

Hawk‚Äëi supports multiple LLM providers for AI‚Äëpowered reasoning. You can enable AI analysis with the --ai flag and specify the model and API key via command line, environment variables, or configuration file.

Command‚ÄëLine Flags

  • --ai ‚Äì Enable AI‚Äëpowered analysis.
  • --ai-model ‚Äì Specify the model (e.g., gemini/gemini-1.5-flash, openai/gpt-4, ollama/codellama).
  • --api-key ‚Äì Provide the API key directly (alternative to environment variables).

Environment Variables

You can set API keys using the following environment variables (recommended for security):


    export GEMINI_API_KEY=your_key_here      # Google Gemini
    export OPENAI_API_KEY=your_key_here      # OpenAI
    export ANTHROPIC_API_KEY=your_key_here   # Anthropic Claude

Local Models with Ollama

To use local models via Ollama, specify the model as ollama/<model_name>, e.g.:

hawki scan ./contracts --ai --ai-model ollama/codellama

Ensure Ollama is running locally and the model is pulled.

Fallback Behavior

If AI is disabled (--ai not used), Hawk‚Äëi automatically falls back to rule‚Äëprovided explanation and impact templates. This guarantees that every finding includes a clear description and remediation guidance, even without an LLM.

Example: Full AI‚ÄëPowered Scan

hawki scan ./my-project --ai --ai-model openai/gpt-4 --api-key sk-... --format pdf --sandbox

Vulnerability Library (30 Rules)

v0.7.0 expands coverage to 30 vulnerabilities, each with:

  • Detection rule (Python, dynamically loaded)
  • Attack script (for critical/high, in attack_scripts/)
  • Fix template (JSON in remediation_engine/templates/)
  • Explanation & impact templates

Categories include reentrancy, access control, oracle manipulation, flash loans, signature replay, and more. See the rules directory for the full list.

Deployment modes

  • Local ‚Äì runs on developer machine, everything in‚Äëmemory or local SQLite.
  • Self‚Äëhosted ‚Äì Docker Compose with PostgreSQL, Redis, optional sandbox nodes.
  • Managed cloud ‚Äì (future) ephemeral scanning, no data retention.

Privacy & Security

Hawk‚Äëi is built with privacy as a first principle:

  • All processing can be done locally, code never leaves your machine.
  • When using cloud AI, you control the API keys; prompts are anonymized.
  • Telemetry is opt‚Äëin and contains no identifiable information.
  • Sandboxes are ephemeral and destroyed after each simulation.
  • No source code is ever uploaded to any remote server.
